At hiring process

Web app with Angular and Spring Framework

Because I applied to a "Full-stack software developer" position, the exercise consisted on developing components of a web application from scratch. The minimal requirements did not request to fully implement every component but I decided to implement the frontend, the backend and one of the optional algorithms. The final application had the following features:

  • Appliction that following an API contract displays cities stored in the provided database (MariaDB).
  • Frontend: Angular CLI, NodeJS, RxJS, Material Design.
  • Backend: Java 8, Spring Boot, REST, Hibernate, Maven. High coverage of unit tests.
  • Algorithm: to find the most permutable city, which is the city giving the highest number of real words from a dictionary stored on the same database (following some rules, like: cities whose name has length 7, permutations with 5,6 and 7 letters, etc.)

Operating systems

► Studied operating system foundations (e.g. processes, protection mechanisms, system calls, structure, classification, real-time).

► Studied management of processor (e.g. Process Control Block, process states, threads, scheduler, scheduling algorithms, signalling). Focus on UNIX (Pthreads) and Win32 threads.

► Studied concurrent computing and concurrence tools in Unix (e.g. concurrence problems, shared memory, mutex, semaphores, conditional variables, message queuing, pipes, signals).

► Studied memory management (e.g. virtual pages), input/output architectures and file systems (Windows and Unix based systems)

► Installed an Unix operating system (FreeBSD), learnt the environment (e.g. commands, structure) and developed a simple application within this environment using C language (example of tools used: bash, sh, vi, gcc, GNU make, gdb). Finally, practiced management of processes.

► Using the previous system, developed and tested a web server that has a variable number of threads and communicates with other processes (CGI). Tools used: C language, Pthreads, system signals, pipes, lynx.

► Developed and tested a multithreaded cache web server using Java and Eclipse. Tools used: Eclipse, several web navigators, communication and synchronization tools offered by Java (queues, mutex, condition variables). Throughout the project, analysed and solved concurrence problems that appear in this sort of application.

Virtual instrumentation (data acquisition systems fundamentals)

► Implemented control system of an oven (it comprises: DAQ, heater, fan; tools for monitoring and controlling sensors, actuators and signals; tools for storing and getting back historical data). Networks and services ubiquitous and secure

► Conceived, designed, implemented, simulated and tested control system for greenhouses or similar environments, employing a Wireless Sensor Network (WSN). The system comprises a sink, a server and several Sun SPOT motes and must manage several types of measurements, alarms and commands; as well as do not fail within a real environment (using physical nodes). I lead the project within a team of 4 people. Due to human resource and knowledge issues, I did major parts: trained 2 of my workmates in Java, PDU structure and system design; designed the software architecture from low-level detail to high-level and coded it in Java. All of us performed the requirement analysis, the general design of the system (e.g. PDU structures and listening ports), the definition of the validation tests, the generation of documentation and the presentation of the system. Tools employed: UML, NetBeans, Java (J2ME).
